# AI Agent Guidance - Project Swarm

This file provides decision heuristics for AI agents using Swarm MCP tools alongside Antigravity's built-in tools.

## Tool Selection

### Code Search

| Task | Recommended Tool | Why |
|------|------------------|-----|
| Exact symbol: `UserModel` | `search_codebase(keyword_only=True)` | Auto-Pilot optimization (~1ms) |
| Conceptual: "auth logic" | `search_codebase` | Semantic understanding |
| Regex pattern | `grep_search` | Regex support only |
| Deep analysis | `retrieve_context` | AST + PageRank |

### Code Modification

| Task | Recommended Tool | Why |
|------|------------------|-----|
| Simple edits | `replace_file_content` | Direct control (Verify first!) |
| Refactoring | `process_task` | OCC Conflict Detection |
| Debugging | `process_task` | Ochiai SBFL |
| Verification | `process_task` | Z3 Verifier |

### File Operations

| Task | Recommended Tool |
|------|------------------|
| View file contents | `view_file` (Verify content before editing) |
| Find files | `find_by_name` |
| Index codebase | `index_codebase` |

## Decision Flowchart

```
Need to find code?
├─ Looking for symbols/concepts?
│  └─ YES → search_codebase (Swarm handles optimization)
├─ Need regex pattern?
│  └─ YES → grep_search
└─ Need full architecture?
   └─ YES → retrieve_context

Need to modify code?
├─ Simple edit?
│  └─ YES → view_file (verify) → replace_file_content
└─ Complex/Refactor/Debug?
   └─ YES → process_task (Routes to specialized worker)
```

## Swarm Tool Heuristics

### search_codebase
- `keyword_only=True`: ~1ms, exact matches
- Default (semantic): ~240ms, conceptual understanding
- Use keyword for: symbols, function names, class names
- Use semantic for: "error handling", "authentication flow"
- **NEW:** Runtime hints auto-suggest keyword_only when symbol detected

### index_codebase
- Run once on first setup
- Re-index after >10 new files
- Providers: auto (recommended), gemini, openai, local

### retrieve_context
- Slower (~500ms-2s) but deeper than search
- Use when search results are incomplete
- Good for: architecture analysis, dependency graphs

### process_task
- Routes to specialized algorithms automatically:
  - "refactor" → OCC (conflict detection)
  - "debug" → SBFL (fault localization)
  - "verify" → Z3 (formal verification)
  - "merge" → CRDT (collaborative editing)
- Best practice: be specific, include context

## Multi-Tool Workflows

### Discovery → Analysis → Modification
```
1. grep_search("auth")           # Quick overview
2. search_codebase("auth logic") # Semantic discovery
3. retrieve_context("auth flow") # Deep analysis
4. process_task("Refactor...")   # Modification with algorithms
```

### Debugging Workflow
```
1. Run tests → identify failure
2. process_task("Debug test_login failure in test_auth.py")
3. Swarm uses Ochiai SBFL to find suspicious lines
4. Fix identified code
5. Re-run tests to verify
```
